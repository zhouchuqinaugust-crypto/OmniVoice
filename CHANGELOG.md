# Changelog

## 0.0.2 - 2026-04-08

This release focuses on making OmniVoice feel more like a usable product while keeping the Horizon Client remote-desktop insertion path intact.

### Highlights

- Added offline audio-file transcription from local recordings to `.txt`.
- Added a drag-and-drop audio transcription window from the macOS menu bar.
- Added long-recording chunking for hour-plus files, with progress shown in the UI.
- Added support for common non-WAV inputs such as `m4a`, `mp3`, and `mp4` through `ffmpeg` when available, with a macOS `afconvert` fallback.
- Added best-effort speaker labels for file transcription when `pyannote.audio` and a Hugging Face token are configured.
- Added a real macOS app icon and simplified the menu bar into a compact waveform icon.
- Reorganized the menu bar items into `Ask`, `Library`, and `Tools` groups.

### Reliability

- Configuration saves now use an atomic write path and keep a `.bak` backup of the previous file.
- History writes are serialized to avoid JSONL corruption from concurrent dictation and Ask operations.
- History is capped to the latest 1,000 events to avoid unbounded growth.
- Short dictation STT processes now time out after 45 seconds instead of spinning indefinitely.
- Process cancellation no longer relies on the previous delayed `0.2s` interrupt/terminate fallback.
- File transcription no longer silently drops whole chunks because of short-dictation confidence filtering; truly silent or empty chunks are reported as warnings.
- The long-file WAV reader now supports both standard PCM WAV and `WAVE_FORMAT_EXTENSIBLE` PCM files generated by macOS tools.

### UX

- Recording and transcription status are clearer in the menu bar icon.
- File transcription now reports chunk progress such as `Transcribing chunk 1/3`.
- User-facing errors are translated into clearer messages for common STT, decoding, and insertion failures.
- The existing clipboard-based remote paste path is intentionally preserved because it is important for Horizon Client reliability.

### Performance

- Audio preprocessing now uses `scipy.signal.lfilter` when available and falls back to the previous pure-Python implementation when SciPy is not installed.

### Notes

- Whisper and MLX models are not included in the repository or release artifacts. MLX models are downloaded locally by Hugging Face on first use.
- A packaged app asset is not attached yet because the current local bundle can include large local STT model files. For now, build from source and configure the local MLX environment as described in the README.
